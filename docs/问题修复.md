# ClipFlow 问题修复

## 1. `electron-vite` 主进程入口缺失

### 问题现象

执行 `npm run dev` 时提示：

```text
An entry point is required in the electron vite main config
```

### 根因

`electron.vite.config.ts` 中仅配置了 `outDir`，没有为 `main` 和 `preload` 显式声明入口文件，`electron-vite` 无法识别打包起点。

### 处理

- 为 `main` 增加 `build.lib.entry`
- 为 `preload` 增加 `build.lib.entry`
- 为 `renderer` 增加 `build.rollupOptions.input`

### 验证

- `npm run dev` 不再报入口缺失
- `npm run build` 可以正常输出 `out/main`、`out/preload`、`out/renderer`

## 2. preload 脚本加载失败导致白屏

### 问题现象

Electron 窗口启动后页面空白，DevTools 中提示：

```text
Unable to load preload script
Cannot use import statement outside a module
```

### 根因

preload 默认产物为 ESM，Electron 当前按普通 preload 脚本执行，导致 `import` 语法无法直接运行。

### 处理

- 将 preload 构建格式改为 `cjs`
- 将输出文件固定为 `index.js`
- 主进程 preload 路径同步指向 `../preload/index.js`

### 验证

- preload 不再报模块格式错误
- `window.clipflow` 可在渲染进程中正常访问
- 页面可正常渲染

## 3. 桌面窗口缺少调试入口

### 问题现象

桌面程序默认没有可直接使用的 `F12` 调试入口，排查白屏和运行期错误不方便。

### 根因

主进程未在开发环境自动打开 DevTools，也未注册常见调试快捷键。

### 处理

- 开发环境自动打开 DevTools
- 支持 `F12`
- 支持 `Ctrl + Shift + I`

### 验证

- `npm run dev` 后可自动弹出 DevTools
- 快捷键可正常唤起/关闭调试面板

## 4. Windows 缓存目录权限告警

### 问题现象

终端中仍会看到如下报错：

```text
Unable to move the cache: 拒绝访问。 (0x5)
Unable to create cache
Gpu Cache Creation failed: -2
```

### 当前判断

该问题更像本机缓存目录权限或占用问题，当前不影响 `npm run dev` 和 `npm run build`，也不属于当前白屏主因。

### 当前处理

- 先记录为已知问题
- 暂不阻塞当前功能开发

### 后续计划

- 单独排查缓存目录位置、权限和占用情况
- 若有必要，再补定向规避方案
