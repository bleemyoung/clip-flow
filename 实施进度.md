# ClipFlow 实施进度

## 当前总览

截至 `2026-04-13`，项目已完成基础壳子搭建，并补齐阶段 3 的剪贴板历史 MVP。当前可以本地启动、可构建，剪贴板历史已经从 mock 数据切换为真实数据链路。

## 阶段进度

### 阶段 1：项目初始化

状态：已完成

- 已初始化 `Electron + Vue3 + TypeScript + Vite` 工程
- 已建立 `main / preload / renderer / shared` 目录结构
- 已补基础类型与常量
- 已跑通 `npm run dev`

### 阶段 2：基础壳子

状态：已完成

- 已完成单窗口主界面
- 已完成 `Clipboard / Transform / Snippets` 三模块切换
- 已完成基础页面骨架和 mock 展示

### 阶段 3：剪贴板历史

状态：已完成 MVP

已完成：

- 在 `main` 层接入文本剪贴板轮询
- 轮询间隔固定为 `1000ms`
- 本地持久化到 `app.getPath('userData')/clipboard-history.json`
- 完成历史记录入库与应用重启恢复
- 完成重复文本去重，重复时仅刷新原记录时间
- 完成普通历史上限 `50`
- 完成置顶 / 取消置顶
- 完成复制回填系统剪贴板
- 完成删除单条记录
- 完成清空普通记录
- 完成 `preload` 受控 API 与渲染层真实数据联动

当前固定规则：

- 仅支持文本剪贴板
- 空字符串和纯空白文本不入库
- 置顶记录不受普通记录上限裁剪影响
- 清空操作默认只清空未置顶记录
- 应用主动复制回填时会忽略本次写入，避免立即重复入库
