# ClipFlow 后续计划

## 当前优先级

### 1. 文本转换补齐交互

- 接入转换结果复制
- 梳理错误提示与状态反馈
- 视复杂度决定是否抽离模块状态

### 2. 常用片段 CRUD 与持久化

- 完成新增、编辑、删除、复制
- 接入本地存储
- 保证应用重启后数据仍存在

### 3. 工程化收尾

- 补基础 lint
- 配置应用图标
- 评估打包分发方式
- 预留 `CI/CD` 接入点

### 4. 问题收口

- 继续排查当前环境下的构建问题
- 重点确认 Node 版本与依赖链兼容性
- 将排查过程和结果同步到问题修复文档

## 已完成阶段 3 结论

剪贴板历史 MVP 已经闭环，当前规则固定如下：

- `main` 轮询系统文本剪贴板
- 轮询间隔固定为 `1000ms`
- 使用本地 JSON 持久化
- 重复文本不新增，只刷新旧记录时间
- 普通历史上限 `50`
- 置顶记录不参与普通记录裁剪
- 支持复制回填、置顶 / 取消置顶、删除单条、清空普通记录

## 推进原则

- 优先闭环，不扩功能
- 优先最小可运行实现，不提前做复杂抽象
- 以后续实际使用反馈决定是否继续加搜索、标签、快捷键等能力
