# ClipFlow 后续计划

## 当前优先级

### 1. 剪贴板历史闭环

- 在 `main` 层接入文本剪贴板轮询
- 完成连续重复去重
- 完成置顶、删除、清空普通记录
- 完成复制回填系统剪贴板
- 接入本地持久化

### 2. 文本转换补齐交互

- 接入转换结果复制
- 梳理错误提示与状态反馈
- 视实际复杂度决定是否抽离模块状态

### 3. 常用片段 CRUD 与持久化

- 完成新增、编辑、删除、复制
- 接入本地存储
- 保证应用重启后数据仍存在

### 4. 工程化收尾

- 补基础 lint
- 配置应用图标
- 评估打包分发方式
- 预留 `CI/CD` 接入点

### 5. 问题收口

- 继续排查 Windows 缓存目录权限告警
- 将排查过程和结果同步到问题修复文档

## 推进原则

- 优先闭环，不扩功能
- 优先最小可运行实现，不提前做复杂抽象
- 以当前项目结构为基础推进，不做无关重构
